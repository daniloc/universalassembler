# Session — 2026-06-05 — Genesis

The session that created UniversalAssembler v0. Captured so the reasoning travels
with the code.

## How it started

User asked about **Gall's Law** ("a complex system that works evolved from a simple
system that worked; a complex system designed from scratch never works"). Then,
deliberately invoking that principle, asked to:

> Create a TypeScript DSL for interpretable software systems development. The top
> level primitive is the Specification(.ts). It explains the top level of the
> software. Specs have subassemblies schematically described on declaration.
> Subassemblies are themselves of the Specification format. The pattern continues
> fractally until the software is defined.

(Originally built in a temp folder `interpretable-spec-dsl/`, then moved here into
`UniversalAssembler/` at user's request.)

## Design framing

Hold the DSL itself to Gall's Law: ship a **simple system that works**, then grow
it. So v0 is deliberately minimal and dependency-free (runs on Node 23.6+ native
TS type-stripping, `node demo.ts`).

## Decisions made (with rationale)

1. **Specs are declarative data, not behavior.** A `Spec` describes; it doesn't run.
   Interpretation is a separate pass. Keeps the primitive tiny and allows many
   interpreters (outline, coverage, future codegen/diagram) over one data model.

2. **The core distinction is schematic vs. elaborated**, not done vs. not-done.
   `part(name, role)` = schematic sketch (✎, a legitimate named promise).
   `part(name, role, () => Spec)` = elaborated (●, recurses — the fractal step).
   This makes incremental growth first-class instead of a TODO comment.

3. **The elaborator is lazy** (`() => Spec`). Enables top-down authoring (declare a
   part before its spec exists) and tolerates cyclic references at module load.

4. **`contract` is plain text in v0.** Obvious v1 is a real TS type so the spec and
   the implementation can't drift — but that's complexity to grow into, not start with.

5. **The frontier is the product.** `frontier()` returns all schematic stubs = the
   backlog. The system is fully interpretable before the frontier empties; it's
   never broken, only less complete. This is the Gall's-Law payoff made concrete.

## What was built

- `src/spec.ts` — `Spec`, `Subassembly`, `specify()`, `part()`, `isElaborated()`.
- `src/interpret.ts` — `walk()` (DFS node stream, cycle-guarded), `outline()`,
  `frontier()`, `coverage()`.
- `examples/link-shortener.spec.ts` — LinkShortener grown 3 levels (Encoder, Store,
  Redirector elaborated; Analytics / CollisionGuard / ReadCache left schematic).
- `demo.ts` — prints outline + coverage + frontier.
